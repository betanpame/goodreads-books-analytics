## Task 03 â€“ Create Portfolio Readiness Checklist

### Objective

Define a **checklist** to confirm the project is ready to showcase in your portfolio.

### Outputs

- A `docs/portfolio-checklist.md` file with items you can tick off.

### Steps

1. **List key areas to review**

   - Example areas:
     - README and documentation.
     - Notebooks and code quality.
     - SQL scripts and database.
     - Data pipeline and tests.
     - Visualizations and figures.

2. **Create checklist items under each area**

   - For example, under README:
     - "[ ] Project overview clear and up-to-date".
     - "[ ] Setup and run instructions tested".
   - Under notebooks:
     - "[ ] Key notebooks run from top to bottom without errors".

3. **Add a final overall check**

   - Example:
     - "[ ] I would be comfortable walking through this project live in an interview".

4. **Use the checklist**
   - Once written, go through each item and mark progress.
   - Note any remaining gaps as future improvements.

### Checklist

- [ ] `docs/portfolio-checklist.md` created.
- [ ] Checklist items written for all key areas.
- [ ] Final overall readiness item included.
- [ ] Checklist used to assess current state.
